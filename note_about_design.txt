## 为什么 Modules 的删除/更新在 `Modules/routes.js`？

因为 **Modules 是独立的资源**，不一定属于某个特定的 Course。所以：

- `DELETE /api/modules/:moduleId` - 直接操作 module
- `PUT /api/modules/:moduleId` - 直接操作 module

但是创建和查找 modules 是在 Course 的上下文中：
- `POST /api/courses/:courseId/modules` - 为特定课程创建 module
- `GET /api/courses/:courseId/modules` - 获取特定课程的 modules

## 对于 Assignments，你应该怎么放？

根据你的需求和常见的 REST API 设计，我建议：

### 选项 1：所有 Assignment 路由都放在 `Assignments/routes.js`
```js
// Assignments/routes.js
app.post('/api/assignments', createAssignment);
app.get('/api/assignments', findAllAssignments);
app.get('/api/assignments/:assignmentId', findAssignmentById);
app.put('/api/assignments/:assignmentId', updateAssignment);
app.delete('/api/assignments/:assignmentId', deleteAssignment);

// 如果需要按课程查找
app.get('/api/courses/:courseId/assignments', findAssignmentsForCourse);
```

### 选项 2：混合方式（类似 Modules）
```js
// Courses/routes.js - 课程相关的 assignment 操作
app.post('/api/courses/:courseId/assignments', createAssignmentForCourse);
app.get('/api/courses/:courseId/assignments', findAssignmentsForCourse);

// Assignments/routes.js - 独立的 assignment 操作
app.get('/api/assignments/:assignmentId', findAssignmentById);
app.put('/api/assignments/:assignmentId', updateAssignment);
app.delete('/api/assignments/:assignmentId', deleteAssignment);
```

---

## 我的推荐：选项 1

把所有 Assignment 相关的路由都放在 `Assignments/routes.js`，因为：
1. **更清晰** - 所有 assignment 操作在一个地方
2. **更容易维护** - 不用在多个文件间跳来跳去
3. **更符合 RESTful 设计** - 按资源类型组织
